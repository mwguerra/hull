1 · Bridge call

Synchronous ping(text) → C++ echoes a JSON result.


        

      


      
      

        
2 · Settings (encrypted at rest)

        
Two-way state persisted by C++ (AES-256-GCM). C++ pushes a
          settings:changed event after every write.

3 · Credentials (write-only)

        
Stored in the OS keychain. Never returned to JS — only a boolean check.

        

          

          

        

        
        
        

          
          
          
        

        

      


      
      

        
4 · HTTP (TLS, from C++)

        
Runs on a C++ worker thread (cpp-httplib + OpenSSL). Adds a
          keychain Bearer token if one exists for the host.
          httpbin.org/anything echoes both verbs.

        
        
        
        
        

          
          
        

        

      


      
      

        
5 · Printers

        
Discover (Winspool / CUPS), then print a test message — a text document
          that works with any printer, incl. Microsoft Print to PDF. Receipt buttons send raw
          ESC/POS for thermal printers (spooler or TCP port 9100).

        

        
        
        
        
        

          
          
        

        
        

          
          
        

        

        

      


      
      

        
6 · Notes (SQLite)

        
Parameterized SQLite in the C++ backend, stored in the per-user app dir.
          migrate sets up the schema once; add/delete persist across restarts.

7 · Files (uploads)

        
Stored in the per-user app dir through the secure layer; names are
          sanitized (no path traversal). Pick a file to upload, then view or delete it.
